Project Description

In this project you write code that simulates a game of Nim. This is a well known game with a number of variants. The following variant has an intersting winning strategy. Two players alternatively take marbles from a pile. In each move, a player chooses a how many marbles to take. The player must take at least one but at most half of the marbles. Then the other player takes a turn. The player who takes the last marble loses. Following is an example of a Game of Nim when it is played with marbles between 10 and 100. It is given to you so you can play the game with pencil and paper to understand how the game is played. Of course, your program will be more general - the number of marbles could be any positive integer. See the output file with sample runs

Generate a random integer between 1 and 100 to denote the initial size of the pile. Generate a random integer between 0 and 1 to decide whether the computer or the human takes the first turn. Generate a random integer between 0 and 1 to decide whether the computer plays smart or stupid. In the stupid mode, the computer simply takes a random legal value (between 1 and n/2) from the pile whenever it has a turn. In the smart mode, the computer takes off enough marbles to make the size of the pile a power of 2 minus 1 - that is, 3, 7, 15, 31 or 63. That is always a legal move except when the size of the pile is currently one less than a power of two. In that case, the computer makes a random move.

You will note that the computer cannot be beaten in the smart mode when it has the first move, unless the pile size happens to be (a power of 2 - 1) such as 15, 31, 63, 127, 255, 511... Of course, the human player who has the first turn and knows the winning strategy can win against the computer.

Background

The project contains two classes – GameOfNim.java and Project4.java. The Project4.java is given to you and it represents a driver module for GameOfNim.java class. You will be writing code for GameOfNim.java. Project4.java has the working code to play a single game. You need to modify Project4.java so it allows the user to play multiple games in one run. Procedure

  • Go over Project4.java and the sample runs. Read the project description and understand what needs to be done. It will be helpful to play the game with a pencil and paper and trace the sample runs.
  • The GameOfNim.java class has the following public interface consisting of two public methods:
    1. The constructor takes minimum and maximum size of the pile (of marbles). This is the header for the constructor.
      public GameOfNim(int min, int max). Note that min and max could be any legal integers where min < max. You can assume that min is less than or equal to max. For example, min could be 200 and max could be 5000. Or, min could be 1000 and max could be 12000. You start the game by first generating a random integer between min and max. You need to figure out what instance variables you need and what the constructor is supposed to do.
    2. play method that simulates the game itself and prints appropriate messages on the screen. This is the header for the play method public void play(). Since the number of marbles could be any integer, you simply cannot hardcode the number of marbles to be removed by the computer when it is playing in the smart mode.
